# Порівняння алгоритмів DFS та BFS для пошуку шляху в графі

## Вступ

У цьому завданні було реалізовано два алгоритми пошуку шляху: **DFS** (Depth-First Search — пошук у глибину) та **BFS** (Breadth-First Search — пошук у ширину) для моделі транспортної мережі метрополітену.
Мета: знайти шлях від станції **"Академмістечко"** до станції **"Либідська"**.

## Результати виконання

### 1. Алгоритм DFS (Пошук у глибину)

**Отриманий шлях:** `['Академмістечко', ..., 'Театральна', 'Золоті Ворота', 'Палац Спорту', 'Площа Українських Героїв', 'Олімпійська', 'Либідська']`

**Характеристика:**
DFS намагається рухатися "вглиб" по гілці якомога далі, перш ніж повернутися назад. У нашому випадку, дійшовши до станції пересадки (наприклад, "Театральна"), він може "пірнути" на іншу гілку (Зелену), пройти її до кінця або до іншої пересадки, і лише потім потрапити на потрібну Синю лінію.

### 2. Алгоритм BFS (Пошук у ширину)

**Отриманий шлях:** `['Академмістечко', ..., 'Хрещатик', 'Майдан Незалежності', ..., 'Либідська']`

**Характеристика:**
BFS досліджує граф "шарами" (спочатку сусідів на відстані 1 крок, потім на відстані 2 кроки і т.д.). Тому він гарантовано знаходить шлях з **мінімальною кількістю переходів** (ребер).

## Висновки та порівняння

1.  **Оптимальність:**

    - **BFS** завжди знаходить найкоротший шлях у невиваженому графі (графі, де відстань вимірюється кількістю кроків). У нашому прикладі це найлогічніший маршрут для пасажира метро.
    - **DFS** не гарантує знаходження найкоротшого шляху. Він просто шукає _будь-який_ шлях, який веде до цілі.

2.  **Принцип роботи:**

    - BFS працює як "хвиля", що рівномірно розходиться від старту.
    - DFS працює як "лабіринт", де ми йдемо до упору, а якщо вперлися в тупик — повертаємося на крок назад.

3.  **Застосування:**
    - Для побудови маршрутів на карті (GPS, метро) краще підходить **BFS** (або його вдосконалення — алгоритм Дейкстри), оскільки нас цікавить найкоротша дорога.
    - **DFS** частіше використовується для повного обходу графа, перевірки зв'язності або пошуку циклів, де довжина шляху не є критичною.
